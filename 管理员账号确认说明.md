# 管理员账号邮箱确认说明

## 问题
登录管理员账号 `leilei` 时提示需要邮箱确认。

## 解决方案

### 方法1：在 Supabase Dashboard 中手动确认（推荐，最快）✅

1. 登录 Supabase Dashboard
   - 访问：https://supabase.com/dashboard
   - 选择你的项目

2. 进入用户管理
   - 点击左侧菜单 **Authentication** -> **Users**

3. 找到管理员账号
   - 在用户列表中搜索 `leilei@admin.com`
   - 或者按邮箱排序查找

4. 确认用户
   - 找到该用户后，点击用户行右侧的 **"..."** 菜单
   - 选择 **"Confirm user"** 或 **"Confirm email"**
   - 或者直接点击用户行，在详情页面点击 **"Confirm"** 按钮

5. 完成
   - 确认后，用户就可以直接登录了
   - 不需要再查收邮件

### 方法2：查收邮件确认

1. 检查邮箱（包括垃圾邮件文件夹）
2. 查找来自 Supabase 的确认邮件
3. 点击邮件中的确认链接
4. 完成确认后即可登录

### 方法3：在 Supabase 设置中禁用邮件确认（开发环境）

如果这是开发环境，可以在 Supabase 设置中禁用邮件确认：

1. 进入 Supabase Dashboard
2. 点击 **Authentication** -> **Settings**
3. 找到 **"Email Auth"** 部分
4. 取消勾选 **"Enable email confirmations"**（或设置为特定邮箱不要求确认）
5. 保存设置

**注意**：生产环境建议保持邮件确认功能开启。

## 快速确认步骤（方法1详细版）

```
1. 打开 https://supabase.com/dashboard
2. 选择项目
3. Authentication -> Users
4. 搜索 leilei@admin.com
5. 点击用户 -> 点击 "Confirm" 按钮
6. 完成！现在可以登录了
```

## 验证确认是否成功

确认后，可以：
1. 在应用登录页面输入 `leilei` 和密码 `123456`
2. 应该能够成功登录，不再提示邮箱确认

## 常见问题

**Q: 找不到用户怎么办？**
A: 确保用户已经注册。可以在 Users 页面查看所有用户列表。

**Q: 确认后还是无法登录？**
A: 检查：
- 密码是否正确（123456）
- 用户名是否正确（leilei 或 leilei@admin.com）
- 清除浏览器缓存后重试

**Q: 可以批量确认用户吗？**
A: 可以，在 Users 页面可以批量选择用户并确认。
